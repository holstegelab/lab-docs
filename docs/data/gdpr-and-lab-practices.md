# GDPR and Lab Practices

We work with human genomic data. Handle all data as controlled data unless explicitly stated otherwise.

## Mandatory rules

- No raw sequencing data on personal laptops
- No Dropbox or Google Drive for controlled data
- No sequencing data by email
- Use approved SURF infrastructure
- Follow university GDPR training requirements

If unsure, ask the PI or a senior lab member before transferring or processing data.

## Operational good practices

Do:

- Use structured directories
- Use version control for code and metadata
- Save environment definitions
- Log software versions
- Use Slurm correctly

Avoid:

- Running heavy jobs on login nodes
- Installing tools globally in shared contexts
- Copying large reference genomes per project
- Inconsistent or ad hoc naming conventions
